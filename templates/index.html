    Tentang Aplikasi
    Navigasi Kampus UNIB menggunakan algoritma Dynamic A* dengan optimasi graf dan segmen jalan presisi tinggi:

    🏫 51 POI Utama + 146 Waypoint (Total 197 Node hasil segmentasi)
    🛣 194 Jalan Utama + 14 Segmen Otomatis (Total 208 Edge jalan)
    🗂 Skenario Dinamis kondisi lalu lintas kampus secara real-time
    ⚙ Bobot Edge Dinamis & Deduplikasi Jalur Otomatis

    🟢 Legenda & Multiplier Kondisi Jalan:
    🟢 Normal: ×1.0
    🟡 Sibuk (Busy): ×2.0
    🟠 Berlubang: ×1.6
    🔴 Ditutup (Closed): ∞ (Tidak dapat dilalui)

    👉 Cara Pakai:
    • Pilih Awal & Tujuan: Pilih titik asal dan tujuan pada formulir Rute, lalu klik Cari Rute Tercepat.
    • Bandingkan Skenario: Tekan tombol 📊 Bandingkan untuk menganalisis ETA rute di semua skenario kampus.
    • Edit Bobot Jalan: Aktifkan mode ✏ Edit Bobot Jalan di alat peta, lalu klik jalur jalan langsung pada peta untuk memodifikasi kondisi, jarak, atau menjadikannya satu arah.
    • Navigasi Tabel Jalan: Klik baris jalan manapun di tab Jalan untuk langsung melakukan zoom ke segmen jalan tersebut.
    • Kelola Skenario: Masuk ke tab 🎭 Skenario untuk mengaktifkan, menonaktifkan, atau membuat skenario kemacetan kustom baru dengan deteksi radius.
    • Dinamis Landmark: Geser landmark (Drag-and-Drop POI) secara langsung di peta untuk perubahan rute dinamis real-time.
    Sumber Data Peta
    📍 Tile Map: OpenStreetMap — peta terbuka dunia, dirender via Leaflet.js
    📐 Koordinat Node: Koordinat GPS asli 51 lokasi landmark kampus UNIB (awalnya dicatat manual menggunakan Google Maps sebanyak 29 lokasi utama, kemudian diperluas dan diposisikan ulang secara presisi melalui antarmuka peta interaktif)
    🛤 Geometri Jalan: Polyline koordinat diambil dari dataset OSM & diverifikasi ulang secara manual untuk akurasi sempurna
    🧭 Routing: Jalan baru menggunakan OSRM API untuk mendapatkan geometri yang mengikuti jalur jalan nyata
    📏 Jarak: Dihitung menggunakan formula Haversine dari koordinat GPS asli, bukan garis lurus
